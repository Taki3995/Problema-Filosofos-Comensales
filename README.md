# Criterios Conceptuales Críticos

## 1. Retención y Espera (Hold and Wait)
El diseño del ciclo operativo exige que cada hilo de entrenamiento intente tomar primero el recurso de hardware ubicado a su izquierda y luego el de su derecha. Al obtener el primer componente, el hilo lo retiene indefinidamente mientras se encuentra a la espera de que el segundo componente sea liberado por un nodo vecino. Esta política de asignación secuencial y retención ciega es el principal causante del bloqueo, ya que bloquea hardware muy necesario sin garantizar que el proceso completo pueda iniciar su fase de cómputo.

## 2. Espera Circular (Circular Wait)
La topología lógica del clúster se define como una red circular donde los recursos se comparten entre nodos vecinos. Debido a la simetría del algoritmo de adquisición (todos los hilos toman primero la izquierda), si las N unidades inician su fase de adquisición de manera simultánea, cada hilo asegurará su componente izquierdo sin ningun problema. Por ende, el componente derecho de cada hilo estará retenido por su vecino directo, creando una cadena cerrada de dependencias donde el nodo 0 espera al 1, el 1 al 2, y el nodo N espera al nodo 0. Esta simetría estructural hace imposible que el conflicto se solucione espontáneamente. 

## 3. Ausencia de Expropiación (No Preemption)
El sistema no tiene mecanismos de revocación de acceso o interrupciones forzadas. La arquitectura dice que si un recurso adyacente se encuentra ocupado, el hilo que solicita debe esperar en segundo plano. Los componentes de hardware solo se liberan voluntariamente por la unidad de cómputo una vez que esta ha finalizado exitosamente su simulación y entra en la Fase de Liberación. Como no existe un árbitro central o un límite de tiempo (timeout) que obligue a un hilo a soltar un recurso retenido improductivamente, el sistema colapsa de forma irremediable sin posibilidad de auto-recuperación.
